### Symptoms
- Python `requests.get()` fails with `[Errno 65] No route to host`
- Only happens when process spawned by cron or launchd
- Works perfectly when spawned via SSH

### What Works from Cron/Launchd
```bash
curl -sk https://192.168.100.122/api/config  # ✅ Works
ping 192.168.100.122                          # ✅ Works
```

### What Fails from Cron/Launchd
```python
requests.get("https://192.168.100.122/...", verify=False)  # ❌ No route to host
```

### Attempted Fixes (All Failed)
1. **Network pre-check with curl** — curl passes, Python still fails
2. **Source address binding** — Explicitly bind to 192.168.100.197, still fails
3. **Login shell** (`bash -l`) — No difference
4. **`script` command for PTY** — No difference
5. **Full PATH and environment variables** — No difference
6. **ThrottleInterval in launchd** — No difference

### Theories
- macOS network sandbox for daemon processes?
- Python SSL/OpenSSL build issue on that machine?
- Some network extension or firewall treating Python differently?
- Homebrew Python vs system Python difference?
